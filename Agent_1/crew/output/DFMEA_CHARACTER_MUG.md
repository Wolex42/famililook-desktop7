# DFMEA: Character Mug Feature — FamiliLook
**Date:** 2026-03-30
**Author:** Change Manager Agent
**Scope:** Full user journey from discovery through print fulfilment
**Product:** Character Mug (11oz ceramic, Prodigi GLOBAL-MUG-W)
**Files Analysed:** KeepsakesModal.jsx, compositionEngine.js, characterHeadlines.js, CharacterMugTemplate.jsx, characters/index.js, templateRegistry.js, printProfiles.js, productCatalog.js, cardExport.js, StylePicker.jsx, mugThemes.js

---

## Executive Summary

56 failure modes identified across 12 categories. **4 are Critical (RPN >= 300)**, **11 are High (RPN 150-299)**, and the remainder are Medium or Low. The single highest-risk finding is **FM-001: the KeepsakesModal never passes `occasion`, `recipient`, or `variant` props to CharacterMugTemplate**, meaning the entire composition engine always runs with defaults regardless of which style the user selects. This renders the Heritage/Carnival/Ubuntu styles and all recipient/occasion intelligence non-functional in production.

## Appendix B: Unreachable Code Paths

Due to FM-001 (props not wired), the following code paths are **never executed in production**:

1. All `occasion !== "generic"` branches in compositionEngine
2. All `recipient !== "unknown"` branches in compositionEngine
3. All `variant === "african"` paths in character selection
4. Gift layout (requires occasion or grandparent recipient)
5. Blend layout's loser_parent recipient path
6. Celebration layout's winner_parent recipient path
7. All recipient-specific headline pools in characterHeadlines.js
8. Occasion header display in Gift layout
9. African character PNG loading (except if hardcoded elsewhere)

**Impact**: Approximately 60% of the composition engine's logic is unreachable. The system currently functions as a single-layout (Hero), single-theme (generic), single-variant (default) product with deterministic headline selection based on winner percentage only.
